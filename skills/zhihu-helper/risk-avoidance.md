# 知乎回答风险规避指南

> 本文件是 zhihu-helper SKILL.md 的渐进式披露子文件

### 7.7 企业相关话题风险规避（重要）

涉及企业、品牌、产品时，需要特别注意法律风险。以下是已经踩过的坑和规避策略。

#### 7.7.1 高风险表达类型

| 风险类型 | 高风险表达 | 安全替代 |
|----------|-----------|----------|
| **无依据贬低** | `XX确实落后了`、`XX不行` | `从公开数据看，XX在某个领域还有提升空间` |
| **绝对化断言** | `XX是虚假宣传`、`XX就是...` | `这可能存在概念界定争议`、`部分观点认为...` |
| **品牌拉踩** | `A比B强太多了`、`A乱杀，B落后` | `不同品牌有不同的技术路线和市场策略` |
| **动机揣测** | `搞出格言论维持存在感`、`为了流量` | `这可能反映了企业在传播策略上的考量` |
| **事实捏造** | `XX自己的XX才是...` | 用`据报道`、`公开信息显示`等限定词，或明确说`个人观察` |

#### 7.7.2 企业投诉触发因素分析

根据实际投诉案例，以下内容最容易被投诉：

1. **直接定性**：说某企业「落后」「不行」「虚假宣传」
2. **无数据支撑的对比**：拉踩对比但没有引用来源
3. **动机猜测**：揣测企业行为背后的动机（如「为了流量」「维持存在感」）
4. **情绪化攻击**：用贬低性措辞攻击企业战略或产品
5. **制造对立**：刻意制造品牌对立引发骂战

#### 7.7.3 安全回答框架

**推荐结构**：`[客观背景] + [数据/事实] + [多元视角] + [开放性结论]`

**转化技巧**：

| 原表达 | 转化后 |
|--------|--------|
| 长城新能源确实落后了 | 从销量占比看，长城在新能源领域还处于追赶阶段 |
| 油改电才是虚假宣传 | 从技术架构看，部分车型存在油改电特征，这在业内有一定争议 |
| 搞点出格言论维持存在感 | 这类言论引发了行业讨论，可能与企业传播策略有关 |
| 比亚迪乱杀，长城不行 | 新能源市场竞争激烈，不同品牌进度有差异 |

**软化词工具箱**：
- `可能`、`某种程度上`、`个人观察`
- `从公开信息看`、`据报道`、`数据显示`
- `存在争议`、`有不同观点`
- `还有提升空间`、`处于追赶阶段`

#### 7.7.4 高风险话题判断

如果问题涉及以下内容，需要格外谨慎：

| 高风险话题 | 风险点 | 建议处理 |
|-----------|--------|----------|
| 企业战略/决策 | 可能被认定为「无依据贬低」 | 用「从市场表现看」「公开信息显示」 |
| 产品技术争议 | 可能被认定为「传播虚假信息」 | 明确说「个人理解」「仅供参考」 |
| 行业竞争对比 | 可能被认定为「引战带节奏」 | 避免直接拉踩，说「各有优势」 |
| 企业负面新闻 | 可能涉及名誉权 | 尽量引用官方回应或权威报道 |

#### 7.7.5 回答自查清单

发表前，逐条检查：

- [ ] 有没有对某个企业/品牌做绝对化负面定性？
- [ ] 有没有无数据支撑的对比和拉踩？
- [ ] 有没有揣测企业动机或战略意图？
- [ ] 有没有用贬低性措辞攻击产品或战略？
- [ ] 涉及企业内容是否加了「个人观察」「据报道」等限定？
- [ ] 能否用「公开数据」「市场表现」替代主观判断？
- [ ] 能否把「不行」「落后」改成「还有提升空间」「处于追赶阶段」？

#### 7.7.6 失败案例与修正

**案例：长城汽车相关回答被投诉**

| 原文 | 风险点 | 修正方向 |
|------|--------|----------|
| 长城新能源确实落后了 | 无依据贬低企业战略 | 从销量占比看，长城新能源占比约X%，相比头部企业还有差距 |
| 油改电才是虚假宣传 | 被认定为捏造虚假信息 | 部分车型采用油改电平台，这在业内有一定争议 |
| 搞点出格言论维持存在感 | 揣测企业动机 | 这类言论引发了讨论，可能与企业传播策略有关 |
| 比亚迪在各个赛道乱杀 | 品牌拉踩 | 新能源市场竞争激烈，头部企业表现突出 |

**教训总结**：
1. 涉及企业评价，必须有数据/事实支撑或明确说「个人观察」
2. 避免绝对化断言，用软化词替代
3. 不要揣测企业动机，只谈观察到的事实
4. 品牌对比要中立，避免拉踩

### 7.8 选题风险规避（重要）

回答写作的风险规避（7.7）只是第一步，选题阶段就要避开高风险话题，避免回答本身就成为风险源。

#### 7.8.1 禁止选题清单

以下类型的话题**禁止选择**，即使热榜排名很高也跳过：

| 禁选类型 | 典型关键词 | 风险说明 |
|----------|-----------|----------|
| **法律案件** | 判决、诉讼、起诉、官司、赔偿、法院、判刑、纠纷 | 可能影响司法公正、当事人名誉权 |
| **企业负面** | 倒闭、破产、裁员、欠薪、举报、投诉、起诉公司 | 企业名誉权风险、可能被投诉 |
| **品牌争议** | XX vs XX、谁更好、翻车、塌房、塌台 | 拉踩对比引发投诉风险 |
| **交通事故** | 车祸、追尾、肇事、伤亡、责任认定 | 当事人隐私、司法公正风险 |
| **医疗纠纷** | 医疗事故、医患纠纷、医院责任 | 当事人隐私、医疗执业风险 |
| **人身攻击** | XX人品、XX人品不行、XX道德 | 个人名誉权风险 |
| **隐私曝光** | XX私生活、XX出轨、XX家庭 | 个人隐私权风险 |
| **敏感时政** | 政策争议、官员、体制、敏感话题 | 平台审核风险 |

#### 7.8.2 选题判断流程

从热榜选择问题时，按以下流程判断：

```
1. 检查标题是否包含禁选关键词 → 如果包含，跳过
2. 检查是否涉及具体企业/品牌负面 → 如果涉及，跳过  
3. 检查是否涉及法律诉讼/纠纷 → 如果涉及，跳过
4. 检查是否涉及个人名誉/隐私 → 如果涉及，跳过
5. 通过以上检查 → 可以选题
```

**关键词检测示例**：

```python
# 禁选关键词列表
SKIP_KEYWORDS = [
    # 法律案件
    "判决", "诉讼", "起诉", "官司", "赔偿", "法院", "判刑", "纠纷", 
    "败诉", "胜诉", "庭审", "被告", "原告", "判决书",
    # 企业负面  
    "倒闭", "破产", "裁员", "欠薪", "举报", "投诉", "起诉",
    # 品牌争议
    "翻车", "塌房", "塌台", "谁更好", "vs", "对比评测",
    # 交通事故
    "车祸", "追尾", "肇事", "伤亡", "责任认定",
    # 医疗纠纷
    "医疗事故", "医患", "医院责任",
    # 其他风险
    "人品", "私生活", "出轨", "家庭",
]

def should_skip(title):
    """判断是否应该跳过该选题"""
    for kw in SKIP_KEYWORDS:
        if kw in title:
            return True, kw
    return False, None

# 使用示例
title = "员工上班吃早餐遭开除，最终获赔8000元"
skip, reason = should_skip(title)
if skip:
    print(f"跳过: 包含关键词 '{reason}'")
else:
    print(f"可选: {title}")
```

#### 7.8.3 替代选题策略

遇到高风险话题跳过后，优先选择以下安全话题：

| 安全类型 | 典型话题 | 特点 |
|----------|----------|------|
| **技术科普** | AI、新功能、新技术原理 | 专业性强、争议少 |
| **行业趋势** | 行业发展、市场变化、未来展望 | 宏观分析、不涉及具体企业 |
| **产品体验** | 产品使用体验、功能评价 | 个人体验、中性表达 |
| **生活话题** | 生活技巧、情感困惑、人际关系 | 个人分享、共鸣强 |
| **知识分享** | 历史、文化、科学、教育 | 知识性强、风险低 |
| **职业话题** | 职场经验、求职技巧、行业入行 | 经验分享、有帮助 |

#### 7.8.4 实际选题示例

**跳过案例**：

| 标题 | 跳过原因 | 风险类型 |
|------|----------|----------|
| 员工上班吃早餐遭开除，最终获赔8000元 | 包含"赔偿" | 法律案件 |
| XX公司被起诉虚假宣传 | 包含"起诉" | 企业负面 + 法律案件 |
| 某车企召回车辆，涉及安全隐患 | 企业负面 | 企业名誉风险 |
| A品牌和B品牌哪个更好 | 包含"哪个更好" | 品牌争议 |
| XX翻车了，怎么看 | 包含"翻车" | 品牌争议/名誉风险 |

**可选案例**：

| 标题 | 话题类型 | 安全原因 |
|------|----------|----------|
| DeepSeek 多模态识图功能上线，体验如何 | 技术体验 | 不涉及法律纠纷、无企业负面 |
| AI 行业发展趋势是什么 | 行业趋势 | 宏观分析、不涉及具体企业 |
| 为什么我们总怪父母没抓住房产红利 | 生活话题 | 个人反思、无风险 |
| 如何快速入门某个技术领域 | 知识分享 | 知识性内容、帮助性强 |

#### 7.8.5 边缘情况处理

有些话题看似安全但实际有风险，需要额外判断：

| 边缘话题 | 判断标准 | 处理建议 |
|----------|----------|----------|
| 产品涨价/降价 | 是否伴随负面解读（如"成本大涨被迫涨价"） | 如果只是价格变化，可选；如果有负面定性，跳过 |
| 企业人事变动 | 是否涉及负面（如"被开除""被解雇"） | 如果只是正常变动，可选；如果有纠纷，跳过 |
| 技术路线争议 | 是否涉及具体品牌攻击 | 如果是技术讨论，可选；如果拉踩品牌，跳过 |
| 行业现象讨论 | 是否点名具体企业负面 | 如果是泛泛讨论，可选；如果点名批评，跳过 |

#### 7.8.6 选题自查清单

从热榜选择问题时，逐条检查：

- [ ] 标题是否包含"判决/诉讼/赔偿/法院"等法律关键词？
- [ ] 是否涉及具体企业的负面新闻或争议？
- [ ] 是否涉及品牌对比或拉踩？
- [ ] 是否涉及交通事故/医疗纠纷当事人？
- [ ] 是否涉及个人隐私或名誉攻击？
- [ ] 话题是否属于安全类型（技术/行业/生活/知识）？

**通过检查 → 可选题；任何一项不通过 → 跳过**
